# 🔄 워크플로우 개요

!!! info "학습 안내"
    MDRM의 핵심 기능인 워크플로우를 시각적으로 설계하고 자동화된 IT 프로세스를 체계적으로 관리하는 전체적인 구조와 기능을 학습합니다.


---

## **1. 메인 화면 구성**

워크플로우 메뉴에 진입하면 좌측 트리 메뉴와 연동된 통합 관리 화면이 나타납니다.

1. **트리 메뉴**: 데이터센터와 그룹, 워크플로우의 계층 구조를 관리합니다. 우클릭 컨텍스트 메뉴를 통해 추가/수정/삭제가 가능합니다.
2. **서브 메뉴**: 선택한 자원에 따라 **워크플로우 목록**, **스케줄 관리**, **글로벌 변수 정의** 화면을 제공합니다.
3. **목록 수 및 필터**: 페이지당 표시 개수 설정과 워크플로우 상태(잠금, 실행 여부, 성공/실패 등)에 따른 정밀 필터링을 지원합니다.
4. **일괄 작업 메뉴**: 여러 워크플로우를 선택하여 실행 취소/종료, RTO 일괄 변경, 담당자 변경, 일괄 이동 등을 수행합니다.

---

## **2. 세부 단계**

워크플로우 기능은 다음의 상세 과정으로 차례대로 학습할 수 있습니다.

| 단계 | 주제 | 주요 내용 |
|:---:|:---|:---|
| 1 | [📂 구성 및 그룹 관리](MDRM_워크플로우_구성_관리.md) | 데이터센터, 그룹 및 시스템 그룹의 위계와 관리 기능 |
| 2 | [🎨 프로세스 설계 및 컴포넌트](MDRM_워크플로우_설계_컴포넌트.md) | 편집 화면 구성 및 컴포넌트별 상세 속성 설정 방법 |
| 3 | [📋 변수 활용 및 조건 분기](MDRM_워크플로우_변수_분기.md) | 글로벌/로컬 변수 정의 및 조건문에 따른 동적 흐름 제어 |
| 4 | [🚀 실행 및 모니터링](MDRM_워크플로우_실행_모니터링.md) | 실행 방식(전체/선택), 실시간 로그 및 장애 조치 가이드 |
| 5 | [📊 관리 및 실행 리포트](MDRM_워크플로우_관리_리포트.md) | 변경 이력(버전), RTO 분석 및 엑셀 결과서 추출 |

## **3. 워크플로우 상태 식별**

| 아이콘 | 상태 | 설명 |
|:---:|:---:|:---|
| :material-play-circle:{ .status-blue } | **실행중** | 워크플로우가 현재 진행 중인 상태 |
| :material-check-circle:{ .status-green } | **완료** | 모든 컴포넌트가 성공적으로 종료된 상태 |
| :material-alert-circle:{ .status-red } | **실패** | 실행 중 오류가 발생하여 실패한 컴포넌트가 있는 상태 |
| :material-pause-circle:{ .status-yellow } | **일시중지** | 관리자에 의해 실행이 일시 중단된 상태 |
| :material-stop-circle:{ .status-grey } | **중단** | 특정 지점에서 실행이 강제 중단된 상태 |
| :material-skip-next:{ .status-grey } | **스킵** | 조건에 따라 실행되지 않고 건너뛴 상태 |

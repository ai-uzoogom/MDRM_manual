# 🧩 위젯 활용 가이드

!!! info "학습 안내"
    대시보드를 구성하는 최소 단위인 위젯의 추가, 관리 방법과 시스템 및 워크플로우 상태 파악을 위한 주요 위젯들의 유형별 특징을 학습합니다.


---

## **1. 위젯 추가 및 관리**

### 위젯 추가 방법
* **일반 추가**: '위젯 추가' 탭에서 원하는 위젯을 직접 선택합니다.
* **템플릿 추가**: 특정 목적에 맞춘 위젯 세트를 한 번에 구성합니다. (기본 운영, 재해복구, MDRM 소개 템플릿 제공)
* **가져오기**: 다른 대시보드에서 이미 설정된 위젯을 그대로 복사해옵니다.

### 위젯 관리 및 개인화
* **위젯 배치**: 헤더 영역을 드래그하여 위치를 변경하고, 우측 하단 모서리를 드래그하여 크기를 조절합니다.
* **유형 변환**: 동일 데이터에 대해 차트, 리스트, 요약 등 다양한 UI 형태를 선택할 수 있습니다.
* **스타일 설정**: 폰트, 배경색, 테두리 스타일은 물론 확대 배율(최대 400%)을 위젯별로 다르게 지정할 수 있습니다.

---

## **2. 주요 위젯 종류 소개**

### 🖥️ 시스템 위젯
| 위젯 이름 | 주요 기능 |
|:---|:---|
| **시스템 연결 상태** | Ping 응답을 기반으로 전체 서버의 온라인/오프라인 상태 모니터링 |
| **그룹별 시스템 상태** | 데이터센터 및 그룹 단위로 리소스의 통합 건전성 파악 |
| **시스템 플러그인 현황** | CPU, RAM 등 특정 자원의 상세 실시간 결괏값 표시 |
| **플러그인 차트** | 모니터링 결괏값의 변화 추이를 5초 주기로 그래프화 |

### 🔄 워크플로우 위젯
| 위젯 이름 | 주요 기능 |
|:---|:---|
| **워크플로우** | 워크플로우 맵을 통해 실시간 진행 단계 및 장애 발생 지점 시각화 |
| **워크플로우 진행률** | 설정된 목표 RTO와 실제 소요 시간을 비교하여 진행 상황 표시 |
| **분류별 진행 목록** | 업무별로 워크플로우를 그룹화하여 통합 모니터링 |
| **타임 리코더** | 실행 시점부터의 경과 시간 측정 및 RTO 초과 여부 감시 |
| **워크플로우 실행 단계** | 복잡한 컴포넌트 흐름을 사용자가 정의한 텍스트 위주로 간략히 표시 |

### 📋 점검작업 및 기타 위젯
* **점검작업 요약**: 정기 점검의 준수율과 성공/실패 현황을 데이터센터별로 집계합니다.
* **일일/기간별 스케줄**: 예약된 워크플로우의 실행 예정을 타임라인으로 관리합니다.
* **스케줄 캘린더**: 월간 단위로 워크플로우 실행 계획과 결과를 달력 형태로 파악합니다.
* **자료실/정보**: 공지사항 게시글 확인 및 시스템 소프트웨어 버전을 표시합니다.

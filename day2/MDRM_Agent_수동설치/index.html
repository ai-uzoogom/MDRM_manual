    수동설치 후 가져오기¶

    설치 방식 요약

    보안 정책상 SSH root 접속이 불가능하거나 네트워크가 분리된 환경(DMZ 등)에서 사용하는 방식입니다. 프라이빗 네트워크 환경의 고객사에서 가장 많이 사용되며, 서비스 통신을 위한 443 및 20080 포트만 허용되면 가능합니다.

    사용 시나리오¶

    다음과 같은 경우에 적합합니다:

    • 🔒 보안 정책으로 SSH root 접속이 제한된 경우
    • 🌐 네트워크 분리로 MDRM에서 직접 접근이 불가능한 경우
    • 🔐 일반 계정으로만 접속 가능한 경우
    • 🏢 DMZ 구간의 서버인 경우

    설치 절차¶

    1단계: Agent 설치 파일 준비¶

    1. MDRM 서버에서 Agent 설치 파일 다운로드
    2. 대상 서버로 파일 전송
    # 설치 파일 다운로드 (MDRM 서버에서)
cd /opt/mdrm/agent
ls -l gam_agent_installer.tar.gz

# 대상 서버로 전송
scp gam_agent_installer.tar.gz user@192.168.1.100:/tmp/

    2단계: 대상 서버에서 설치¶

    대상 서버에 로그인하여 설치를 진행합니다.

    # root 계정으로 전환
sudo su -

# 설치 디렉토리로 이동
cd /tmp

# 압축 해제
tar -xzf gam_agent_installer.tar.gz

# 설치 스크립트 실행
cd gam_agent_installer
./install.sh

    3단계: 설치 옵션 설정¶

    설치 스크립트 실행 시 다음 정보를 입력합니다:

    항목 설명 기본값
    설치 경로 Agent 설치 디렉토리 /opt/gam_agent
    MDRM 서버 IP MDRM 서버 주소 -
    MDRM 서버 포트 MDRM 서버 포트 8080
    Agent 포트 Agent 리스닝 포트 20080

    4단계: Agent 시작¶

    # Agent 시작
cd /opt/gam_agent
./bin/start.sh

# 프로세스 확인
ps -ef | grep gam_agent

# 로그 확인
tail -f logs/agent.log

    5단계: MDRM에서 시스템 가져오기¶

    1. MDRM 웹 콘솔 로그인
    2. 시스템 관리 > 시스템 가져오기 메뉴 선택
    3. 대상 서버 정보 입력:
    4. IP 주소: Agent가 설치된 서버 IP
    5. 포트: Agent 포트 (기본 20080)
    6. 인증 키: Agent 설치 시 생성된 키
    7. 가져오기 버튼 클릭

    설정 파일 확인¶

    application.properties¶

    Agent의 주요 설정 파일입니다.

    # MDRM 서버 연결 정보
mdrm.server.ip=192.168.1.10
mdrm.server.port=8080
mdrm.server.https=false

# Agent 설정
server.port=20080
agent.agentId=N0000

# Heartbeat 설정
agent.heartbeat=60000

    설정 파일 위치¶

    # 설정 파일 경로
/opt/gam_agent/config/application.properties

# 설정 파일 편집
vi /opt/gam_agent/config/application.properties

# Agent 재시작 (설정 변경 후)
./bin/restart.sh

    방화벽 설정¶

    Agent와 MDRM 서버 간 통신을 위해 방화벽 설정이 필요합니다.

    Agent 서버 방화벽¶

    # Agent 포트 오픈 (20080)
firewall-cmd --permanent --add-port=20080/tcp
firewall-cmd --reload

# 방화벽 규칙 확인
firewall-cmd --list-ports

    MDRM 서버로의 아웃바운드¶

    # MDRM 서버 포트로 아웃바운드 허용 (8080)
firewall-cmd --permanent --add-port=8080/tcp
firewall-cmd --reload

    설치 확인¶

    Agent 상태 확인¶

    # 프로세스 확인
ps -ef | grep gam_agent

# 포트 리스닝 확인
netstat -nltp | grep 20080

# 로그 확인
tail -f /opt/gam_agent/logs/agent.log

    MDRM 연결 확인¶

    # MDRM 서버로 연결 테스트
telnet 192.168.1.10 8080

# Heartbeat 로그 확인
grep "heartbeat" /opt/gam_agent/logs/agent.log

    MDRM 웹 UI 확인¶

    1. 시스템 목록에서 가져온 서버 확인
    2. 상태가 "정상"으로 표시되는지 확인
    3. 마지막 Heartbeat 시간 확인

    문제 해결¶

    Agent가 시작되지 않음¶

    오류: Agent 시작 실패

    원인: Java 미설치, 포트 충돌, 권한 문제

    해결방법:

    # Java 설치 확인
java -version

# 포트 사용 확인
netstat -nltp | grep 20080

# 권한 확인
ls -l /opt/gam_agent
chown -R root:root /opt/gam_agent

    MDRM 서버 연결 실패¶

    오류: MDRM 서버에 연결할 수 없음

    원인: 네트워크 문제, 방화벽 차단, 설정 오류

    해결방법:

    # 네트워크 연결 테스트
ping 192.168.1.10
telnet 192.168.1.10 8080

# 방화벽 확인
firewall-cmd --list-all

# 설정 파일 확인
cat /opt/gam_agent/config/application.properties

    시스템 가져오기 실패¶

    오류: 시스템을 가져올 수 없음

    원인: 인증 키 불일치, Agent 미실행

    해결방법:

    1. Agent가 실행 중인지 확인
    2. 인증 키 재확인
    3. Agent 로그에서 오류 메시지 확인
    4. Agent 재시작 후 재시도

    장점 및 단점¶

    장점¶

    ✅ 보안 - Root 계정 직접 노출 불필요 - 네트워크 분리 환경에서도 설치 가능

    ✅ 유연성 - 설치 과정을 직접 제어 가능 - 설정 파일 사전 검토 가능

    단점¶

    ❌ 복잡성 - 수동 작업 단계가 많음 - 설정 오류 가능성 존재

    ❌ 시간 - 자동설치보다 시간 소요 - 여러 서버 설치 시 반복 작업 필요

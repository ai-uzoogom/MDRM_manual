## 1. 점검 스케줄 작업
특정 시간이나 주기마다 반복되어야 하는 점검 작업을 자동화 예약합니다.

### 1.1 실행 주기 설정
MDRM은 유연한 스케줄링 옵션을 제공합니다.
- **표준 주기**: 한 번만, 매일, 매주, 매월, 매년
- **크론(Cron) 표현식**: 유닉스/리눅스 표준인 Cron 정규식을 사용하여 더욱 정밀한 주기 설정 가능 (예: `0 0 12 * * ?` -> 매일 정오)

### 1.2 스케줄 상태 모니터링
여러 작업이 묶인 스케줄의 경우, 복합적인 상태 우선순위에 따라 대표 상태가 표시됩니다.
- **실패(Priority 1)**: 라이선스 만료, 중복 실행 등으로 스케줄 작동 실패
- **실행 취소(Priority 2)**: 사용자 요청에 의해 중단
- **실행 중(Priority 3)**: 현재 프로세스 가동 중
- **완료(Priority 4)**: 모든 작업 종료

---

## 2. 알림 관리
점검 결과에 이상이 있거나 특이 사항 발생 시 알림 내역을 조회하고 해결 상태를 관리합니다.

### 2.1 알림 조회 및 필터
- **조회 조건**: 최근 날짜별(1일~한 달) 또는 특정 기간을 설정하여 과거 알림 추적 가능
- **필터 활용**: '미해결' 상태의 알림만 우선적으로 모니터링하여 누락 방지

### 2.2 알림 해결 (Resolve)
발생한 알림을 확인하고 조치를 마친 후 `[해결]` 버튼을 클릭하면 목록에서 사라지며, 대시보드의 알림 수 표시가 갱신됩니다. 이는 실시간으로 관리해야 할 이슈가 남아있는지 확인하는 핵심 지표입니다.

# 🛑 MDRM 정지 및 종료

## docker-compose stop vs down의 차이점

!!! warning "명령어 선택 시 주의"
    용도에 따라 적절한 명령어를 선택해야 합니다.

---

## docker-compose stop

```bash
docker-compose stop
```

| 특징 | 설명 |
|:---|:---|
| 동작 | 컨테이너를 **일시 정지** |
| 데이터 | 컨테이너 설정 및 데이터는 **보존됨** |
| 재시작 | 빠르게 복구 가능 |

!!! tip "사용 시나리오"
    - 잠시 시스템을 중지해야 할 때
    - 서버 재부팅 전
    - 빠른 재시작이 필요한 경우

---

## docker-compose down

```bash
docker-compose down
```

| 특징 | 설명 |
|:---|:---|
| 동작 | 컨테이너를 **완전히 삭제** |
| 네트워크 | 함께 **제거** |
| 볼륨 | 별도로 지정하지 않으면 **유지됨** |

!!! danger "주의사항"
    - 컨테이너가 완전히 삭제되므로 재시작 시 새로 생성됩니다
    - 볼륨 데이터는 기본적으로 유지되지만, `-v` 옵션 사용 시 삭제됩니다

!!! tip "사용 시나리오"
    - 시스템 업그레이드 시
    - 완전한 재설치가 필요한 경우
    - 환경 초기화가 필요한 경우

---

## 명령어 비교 요약

| 명령어 | 컨테이너 | 네트워크 | 볼륨 | 재시작 속도 |
|:---|:---:|:---:|:---:|:---:|
| `stop` | 정지 | 유지 | 유지 | 빠름 |
| `down` | 삭제 | 삭제 | 유지* | 느림 |

*`-v` 옵션 사용 시 볼륨도 삭제됨

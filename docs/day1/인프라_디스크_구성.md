# 💽 디스크 구성

## **DATA DISK 영역 구성**

MDRM 솔루션의 안정적인 운영을 위해 다음과 같은 디렉토리 구조로 구성할 것을 권장합니다.

| 경로 | 설명 | 비고 |
| :--- | :--- | :--- |
| <span style="white-space: nowrap;">`/mdrm/engine`</span> | Docker 컨테이너의 엔진 데이터가 저장되는 영역입니다. | |
| <span style="white-space: nowrap;">`/mdrm/data`</span> | MDRM의 실제 서비스 데이터가 볼륨 매핑되는 영역입니다. | <span style="color: #ef4444; font-weight: bold;">⚠️ 백업 권장</span> |
| <span style="white-space: nowrap;">`/mdrm/package`</span> | MDRM 설치 패키지 및 기타 유틸리티 보관용으로 사용합니다. | |

!!! info "디스크 용량 산정 가이드"
    디스크 사용량은 관리 대상 서버의 규모와 모니터링 플러그인의 설정 주기 등에 따라 달라질 수 있습니다.

    **디스크 사용량 산정 예시 (참고용):**

    - 관리 서버: 20대 기준
    - 모니터링 플러그인: 서버당 5개, 수집 간격 10초 설정 시
    - **예상 결과**: 1년간 약 20GB 내외의 디스크 공간이 소요됩니다.

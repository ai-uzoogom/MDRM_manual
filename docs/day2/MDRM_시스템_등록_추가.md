# ➕ 시스템 등록 및 추가

관제 자원으로 활용할 서버, 스토리지, 네트워크 장비를 신규 등록하거나 이미 에이전트가 설치된 시스템을 가져와 관리 목록에 포함시킵니다.

---

## 1. 시스템 추가 (신규 등록)

새로운 자원을 MDRM 관제 대상으로 등록하는 과정입니다.

### 사전 준비
* [시스템 요구사항](MDRM_Agent_자동설치.md#requirements)에 따른 OS별 환경(SSH, Python, WinRM 등)이 갖춰졌는지 확인하십시오.

### 등록 절차
1. 트리에서 대상 그룹 선택 후 **[시스템 추가]** 클릭
2. **유형 선택**: 서버, 스토리지, 네트워크 중 선택
3. **정보 입력**: 
    * **직접 입력**: 시스템명, IP, 접속 방식(SSH/WinRM), 포트, 계정 정보 입력
    * **일괄 등록**: 엑셀(Excel/CSV) 양식을 다운로드하여 작성 후 업로드
4. **사전 점검**: [사전 점검] 버튼을 통해 접속 가능 여부 및 권한 유효성을 반드시 확인합니다.
5. **생성**: 모든 시스템의 점검이 성공하면 [생성]을 눌러 최종 등록합니다.

---

## 2. 시스템 가져오기 (기존 자원)

이미 GAM 에이전트가 설치되어 있거나 다른 서버에서 관리 중인 자원을 MDRM 서버로 연결합니다.

* **용도**: 에이전트가 사전 수동 설치된 시스템을 등록할 때 사용합니다.
* **필수 정보**: 시스템명, IP, 에이전트 포트(기본 20080), 에이전트 관리자 계정(admin) 정보가 필요합니다.
* **절차**: 시스템 메뉴의 [시스템 가져오기] 기능을 통해 등록 정보를 입력하고 사전 점검을 수행합니다.

---

## 3. 시스템 상세 관리 및 설정

등록된 서버에 대해 추가적인 속성을 정의하거나 관리 작업을 수행합니다.

| 기능 | 설명 |
|:---|:---|
| **수정** | 등록된 시스템의 이름, IP 주소, 에이전트 포트 정보를 변경합니다. |
| **VM 설정** | 가상 머신(VM)인 경우 하이퍼바이저와 연동하여 UUID 정보를 저장하고 제어 기능을 활성화합니다. |
| **LOM 설정** | 물리 서버의 IPMI, iLO, DRAC 등 원격 제어용 장치 정보를 입력하여 관리합니다. |
| **통합 관리 제거** | 에이전트는 유지한 채 MDRM의 관리 자원 목록에서만 제외합니다. |
| **삭제** | 에이전트 언인스톨을 포함하여 시스템 데이터 전체를 삭제합니다. (OS 계정 정보 필요) |

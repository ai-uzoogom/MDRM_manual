# ⚙️ 점검 및 작업 관리

점검 대상을 정의하는 부모(점검)와 실질적인 실행 단위인 자식(작업)의 관계를 설정합니다.

---

## 1. 점검(Inspection) 관리

'점검'은 실질적인 실행 목록을 담는 컨테이너 역할을 수행합니다.

* **기본 정보**: 점검의 명칭과 상세 설명을 정의합니다.
* **알림 설정**: 작업의 시작과 종료 시점에 알림을 보낼 대상자 및 수신 채널을 지정합니다.
* **점검 복사**: 기존 설정을 그대로 복제하여 유사한 점검 환경을 빠르게 구축합니다. (하위 작업 정보까지 일괄 복사됩니다.)
* **잠금 기능**: 불필요한 변경을 방지하기 위해 `편집 잠금` 또는 `실행 잠금`을 적용할 수 있습니다.

---

## 2. 작업(Task) 관리

'작업'은 실제 서버에서 명령(스크립트)을 수행하여 결과를 가져오는 최소 단위입니다.

### 2.1 작업 추가 프로세스
1. **대상 서버 지정**: 점검을 수행할 서버를 <시스템> 메뉴에서 연동합니다.
2. **항목 배치**: 미리 등록된 컴포넌트 형태의 점검 항목을 선택하여 구성합니다.
3. **변수 매핑**: 서버별 특성에 따른 변숫값(IP, ID, 정규식 등)을 입력하여 실행 환경을 완성합니다.

### 2.2 시스템 변수 가이드
스크립트 내에서 직접 사용하거나 변수 입력 시 호출할 수 있는 핵심 시스템 변수입니다.

| 변수명 | 설명 | 활용 예시 |
| :--- | :--- | :--- |
| `ACCOUNT_USERID` | 접속 계정 아이디 | 권한 체크 시 활용 |
| `SYSTEM_IP` | 대상 서버의 IP 주소 | 네트워크 통신 확인 |
| `REMOTE_OUTPUT` | 원격 명령어 실행 결과 | 조건문 판단 근거 |
| `NODE_NAME` | 관리 서버의 명칭 | 리포트 식별자 |

---

## 3. 결과 상태의 이해

작업 실행 시 솔루션은 스크립트의 반환값을 분석하여 다음과 같이 상태를 정의합니다.

* **준수 (OK)**: 정상 범위 내에서 완벽하게 작동 중임을 의미합니다.
* **미준수 (Critical)**: 규정된 임계치를 벗어나거나 비정상 상태임을 의미합니다.
* **실패 (Fault)**: 네트워크 또는 권한 문제 등으로 점검 수행 자체가 불가능한 상태입니다.

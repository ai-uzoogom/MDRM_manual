## 1. 점검작업 개요
일일 점검과 같은 반복적인 시스템 운영 작업은 관리자에게 적잖은 부담을 줍니다. MDRM의 **점검작업** 기능을 사용하면 이러한 반복 업무를 자동화하여 작업자의 부담과 실수를 줄이고, 보다 정밀한 시스템 모니터링이 가능해집니다.

!!! info "점검작업의 핵심 가치"
    - **자동화**: 정해진 스케줄에 따라 시스템 상태를 자동으로 확인
    - **신뢰성**: 수동 점검 시 발생할 수 있는 누락 및 오판 방지
    - **가시성**: 대시보드 요약 화면을 통해 전사 점검 현황을 한눈에 파악

## 2. 요약 화면 (Dashboard)
데이터센터를 클릭하면 나타나는 요약 화면을 통해 점검 현황을 즉시 파악할 수 있습니다.

### 2.1 주요 UI 구성
1. **필터 선택 박스**: 자주 사용하는 점검 조건(작업, 시스템 등)을 저장하고 불러옵니다.
2. **숫자로 보는 현황**:
    - **준수율**: 전체 점검 항목 중 '준수(OK)' 상태의 비율
    - **점검/작업/시스템 수**: 현재 관리 중인 객체의 총량
    - **실행 중**: 현재 실시간으로 돌아가고 있는 작업 수
3. **상태별 필터 버튼**: 미준수, 실패, 준수 상태별로 목록을 필터링하여 확인합니다.
4. **상세 결과 피드**: 선택한 작업의 기본 정보와 세부 실행 메시지를 우측 패널에 즉시 표시합니다.

---

### 3. 점검작업 필터 관리
필터를 사용하면 수많은 작업 중 자신이 관리하는 특정 작업/시스템만 빠르게 확인할 수 있습니다.

| 기능 | 설명 |
| :--- | :--- |
| **필터 추가** | 작업/시스템 선택 후 상단 [저장] 버튼을 클릭하여 이름 지정 |
| **기본 필터 설정** | 특정 필터를 `default`로 지정하면 요약 화면 진입 시 해당 조건이 자동 적용 |
| **필터 수정/삭제** | 기존 필터의 조건을 변경하거나 다른 이름으로 저장, 불필요한 필터 삭제 관리 |

# 🔍 점검작업 개요

!!! info "학습 안내"
    MDRM의 점검작업은 반복적인 시스템 운영 업무를 자동화하여 누락을 방지하고, 직관적인 대시보드를 통해 전사적인 인프라 건전성을 한눈에 파악하는 전체적인 체계를 학습합니다.


---

## **1. 요약 화면 (Dashboard)**

점검작업 메뉴의 데이터센터를 클릭하면 나타나는 요약 화면을 통해 전반적인 점검 현황을 즉시 파악할 수 있습니다.

1. **필터 선택 박스**: 자주 사용하는 점검 조건(작업, 시스템 등)을 저장하고 불러오기 기능을 제공합니다.
2. **숫자로 보는 현황**:
    - **준수율**: 전체 점검 항목 중 '준수(OK)' 상태의 비율을 의미합니다.
    - **점검/작업/시스템 수**: DC 내 관리 중인 객체의 총 수량을 표시합니다.
    - **실행 중**: 현재 실시간으로 가동되고 있는 작업의 수입니다.
3. **상태별 필터 버튼**: 미준수, 실패, 준수 상태별로 목록을 필터링하여 특정 이슈를 빠르게 확인합니다.
4. **상세 결과 피드**: 목록에서 작업을 선택하면 우측 패널에 상세 실행 메시지와 기본 정보를 즉시 표시합니다.

## **3. 점검 결과 상태 식별**

| 상태 | 아이콘 | 설명 |
| :---: | :---: | :--- |
| **준수** | :material-check-circle:{ .status-green } | 점검 결과 "OK"를 반환하여 정상 범위에 있는 상태 |
| **미준수** | :material-alert-circle:{ .status-red } | 점검 결과 "CRITICAL"을 반환하여 조치가 필요한 상태 |

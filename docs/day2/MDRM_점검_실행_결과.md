# 📊 실행 결과 및 분석 리포트

점검 결과를 정교하게 분석하고 조치 내역을 체계적으로 관리하여 시스템 가용성을 보장합니다.

---

## 1. 실행 이력 추적

모든 작업은 '작업 이력' 화면에 영구적으로 기록되어 기술 부채를 추적할 수 있도록 돕습니다.

* **시간 관리**: 시작/종료 시점 및 전체 소요 시간을 기록하여 인프라 부하를 모니터링합니다.
* **실행 모드 식별**: 수동 실행과 스케줄 기반 자동 실행을 명확히 구분하여 표시합니다.
* **준수율 통계**: 서버와 항목 수를 기반으로 한 최종 가용성 지표(%)를 자동 산출합니다.

---

## 2. 상세 결과 분석

수치 이면의 원시 기록(Raw Message)을 확인하여 정확한 원인을 분석합니다.

* **서버별 큐**: 서버 시점에서 실패한 항목들을 전방위적으로 점검합니다.
* **항목별 큐**: 특정 점검 규칙(예: 메모리 부족)이 전사 인프라에서 어떻게 발생했는지 일괄 파악합니다.
* **조치 및 계획(Action Plan)**: '미준수' 상태인 항목에 대해서는 시스템 내에 조치 내역을 기록하여 후속 관리를 강화합니다.
* **메시지 팝업**: 텍스트 형태의 실행 전문을 즉시 열람하여 스크립트 출력값을 확인합니다.

---

## 3. 리포트 자동화 (Excel Export)

필요한 데이터를 문서화하여 외부 보고서나 정기 문서로 활용합니다.

1. **맞춤형 추출**: `서버 중심` 혹은 `점검 항목 중심`의 리포트 중 목적에 맞는 타입을 선택합니다.
2. **엑셀 다운로드**: 필터링된 현재 화면의 정보를 그대로 엑셀 시트로 변환합니다.
3. **오프라인 활용**: 다운로드된 파일은 기업 표준 리포트 양식으로 즉각 활용 가능합니다.

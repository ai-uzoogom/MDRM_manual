# 🔑 계정 관리

시스템 접속에 필요한 ID와 비밀번호를 관리합니다. 계정을 사전 등록해두면 시스템 추가, 워크플로우 실행 시 매번 정보를 입력할 필요 없이 간편하게 선택하여 사용할 수 있습니다.

---

## 1. 계정 생성 및 범위

계정은 생성 위치에 따라 관리 범위와 권한이 구분됩니다.

* **데이터센터 계정**: 데이터센터 내의 **모든 시스템**에 일괄 적용이 가능한 계정입니다. 관리 편의성이 높습니다.
* **개별 시스템 계정**: **특정 시스템**에만 귀속되는 계정입니다. 보안 정책상 특정 서버에만 접근이 필요한 경우 사용합니다.

### 접속 유형 선택
SSH(Linux/Unix), WinRM(Windows), VMWare(Virtualization), Telnet 등 대상 서버의 환경에 맞는 접속 방식을 지정합니다.

---

## 2. 기본 접속 계정 설정

시스템 관리의 핵심 설정으로, 모니터링 및 자동화 작업의 **기본 인증 정보**로 활용됩니다.

* **역할**: 워크플로우 실행, 점검 작업 수행, 시스템 요약 정보 수집 시 기본으로 사용됩니다.
* **설정 방법**: 개별 시스템의 [계정] 메뉴 → [접속 계정 변경] → 기등록된 계정 중 하나를 선택.
* **연결 테스트**: [연결 테스트] 버튼을 눌러 설정된 계정으로 실제 접속이 가능한지 즉시 검증합니다.

---

## 3. 계정 일괄 관리 및 보안

사내 보안 정책 준수를 위해 다수 시스템의 접근 정보를 효율적으로 관리합니다.

### 비밀번호 일괄 변경 (Administrator 전용)
1. **대상 선택**: 비밀번호 변경이 필요한 시스템들을 트리나 검색으로 선택합니다.
2. **엑셀 활용**: 시스템 목록을 엑셀로 다운로드하여 새 비밀번호를 작성한 후 다시 업로드하여 일괄 반영할 수 있습니다.
3. **적용 및 검증**: 새로운 비밀번호를 일괄 적용하기 전 연결 테스트를 통해 정합성을 확인합니다.

### 계정 활용 시나리오
* **활용 1**: 주기적인 패스워드 변경 시, 각각의 워크플로우를 수정할 필요 없이 [계정] 메뉴에서 해당 계정 정보만 변경하면 모두 적용됩니다.
* **활용 2**: 신규 시스템 추가 시 에이전트 설치 권한 계정을 사전 등록된 정보에서 선택하여 오타를 방지합니다.

# 🎨 프로세스 설계 및 컴포넌트

!!! info "학습 안내"
    워크플로우 편집 화면의 구성 요소를 이해하고, 다양한 컴포넌트를 배치하여 업무 시나리오에 맞는 최적의 자동화 흐름을 설계하는 방법을 학습합니다.


---

## **1. 워크플로우 편집 화면 구성**

편집 화면은 크게 4가지 영역으로 구분됩니다.

1. **상단바**: 화면 비율 조정, 테이블 뷰 전환, 컴포넌트 가져오기, 변수 설정 및 저장/취소 기능을 제공합니다.
2. **컴포넌트 메뉴**: 드래그 앤 드롭으로 배치할 수 있는 작업 단위들입니다.
3. **캔버스 영역**: 컴포넌트를 배치하고 연결선으로 흐름을 정의하는 중앙 작업 공간입니다. (Select/Pan 모드 지원)
4. **정보 패널**: 선택한 워크플로우나 컴포넌트의 상세 속성(정보, 진행 속성, 옵션, 결과 처리, 스타일)을 설정합니다.

---

## **2. 주요 컴포넌트 종류**

| 아이콘 | 이름 | 주요 기능 및 용도 |
|:---:|:---|:---|
| :material-cube-outline: | **기본 컴포넌트** | 스크립트 실행, 명령어 전달 등 실제 작업을 수행하는 최소 단위 |
| :material-link-variant: | **링크 컴포넌트** | 다른 워크플로우를 하나의 작업처럼 포함 (재사용성 극대화) |
| :material-rhombus-outline: | **분기 컴포넌트** | 변수 값에 따라 흐름을 여러 갈래로 분기 (IF/ELSE) |
| :material-account-check: | **수동/승인** | 다음 단계 진행 전 사용자의 확인 또는 승인권자의 결재 대기 |
| :material-clock-outline: | **예약/타이머** | 특정 시각(진행 시각) 또는 일정 시간(대기 시간) 경과 후 실행 |

---

## **3. 컴포넌트 상세 설정 (패널 정보)**

컴포넌트를 클릭하면 나타나는 패널에서 다음 5가지 탭을 설정할 수 있습니다.

*   **정보**: 이름, 실행 명령(종류), 연결 서버(Target), 접속 계정, RTO, 담당자 설정.
*   **진행 속성**: 타임아웃, 실패 시 재시도 횟수, 성공 처리 여부, 별도 실행 계정 지정.
*   **옵션**: 실행 전/후 선행/후행 확인 스크립트(Pre/Post Check) 설정.
*   **결과 처리**: 결과 메시지를 가공(정규표현식)하여 특정 변수에 저장.
*   **스타일**: 아이콘 이미지, 폰트 속성, 정밀 위치(좌표) 조정.

---

## **4. 편집 편의 기능**

*   **테이블 뷰**: 모든 컴포넌트를 표 형식으로 일괄 확인하고 수정합니다.
*   **북마크**: 자주 사용하는 설정의 컴포넌트를 저장하여 빠르게 재사용합니다.
*   **자동 정렬**: 수직/수평 정렬 기능을 통해 복잡한 흐름도를 한 번에 정리합니다.
*   **일괄 변경**: 다수의 컴포넌트를 선택하여 담당자, 스타일, 서버 정보를 한 번에 수정합니다.

# 📘 MDRM 교육 매뉴얼 프로젝트 가이드 (AI 협업용)

이 문서는 MDRM 교육 매뉴얼의 일관된 품질과 스타일을 유지하기 위해 작성되었습니다. 다른 AI 도구나 새로운 대화 창에서 작업을 이어갈 때 이 가이드를 반드시 참고하십시오.

---

### **1. 핵심 톤앤매너 (Tone & Manner)**
*   **어투**: 반드시 **친절한 경어체(~합니다, ~입니다)**를 사용합니다. 
*   **대상**: 시스템 엔지니어 및 기술 교육 수강생.
*   **원칙**: 전문적이되 딱딱하지 않게, 핵심을 명확하게 전달합니다.

### **2. 페이지 표준 구조**
모든 마크다운(.md) 문서는 다음 구조를 엄격히 준수합니다.

1.  **파일 명명 규칙**: 
    *   각 DAY의 시작(인덱스) 페이지는 **`MDRM_DAYX_학습_안내.md`**로 명명합니다.
    *   **실무 트러블슈팅** 페이지는 **`MDRM_TS_분류_주제.md`** 형식을 유지합니다. (예: `MDRM_TS_Agent_통신점검.md`)
    *   모든 파일 콘텐츠 제목과 `mkdocs.yml` 내비게이션 레이블을 일치시킵니다.
2.  **메인 제목**: `# [이모지] 주제명` (예: `# 🚀 솔루션 개요`, `# ⚙️ 인프라: OS 튜닝`)
3.  **요약/목표 박스**: 
    ```markdown
    !!! info "학습 안내"
        해당 페이지에서 다루는 핵심 내용과 목표를 사용자 친화적인 어조로 설명합니다.
    ```
4.  **본문 섹션**: `## **1. 섹션 제목**` (번호와 볼드 처리를 통해 구조적 명확성 확보)
5.  **다음 단계 카드 (Next Step)**: 페이지 최하단에 배치하며, 대상 페이지의 성격에 맞춰 제목(예: `... 가이드`, `... 표준`)을 보강합니다.

### **3. 주요 스타일 속성**
*   **안내 박스 (Admonition) 사용 제한**: 과도한 사용은 문서 가독성을 해치므로 아래 규칙을 엄격히 준수합니다.
    *   **사용 조건**: 페이지 상단의 **`!!! info "학습 안내"`** (필수) 외에는 **핵심적인 핵심 내용**이거나 **사용자가 명시적으로 요청한 경우**에만 사용합니다.
    *   **수량 제한**: 한 페이지 내에 안내 박스의 종류(type)가 **3개 이상** 들어가지 않도록 제한합니다. (예: info, tip, warning 3개까지만 허용)
*   **강조/주의**: `!!! danger` (절대 주의), `!!! warning` (주의), `!!! tip` (꿀팁), `!!! example` (예시)
*   **다이어그램**: 프로세스 설명 시 `mermaid` 차트를 적극 활용합니다.
*   **가독성**: 긴 문장보다는 불렛 포인트와 테이블을 선호하며, 코드 블록은 중첩된 박스(!!! example 등) 보다는 마크다운 표준 코드 블록 형식을 권장합니다.

### **4. 기술 및 명명 규칙 (Terminology & Rules)**
*   **Agent 용어 통일**: '에이전트', 'MDRM 에이전트', 'GAM 에이전트'는 예외 없이 **`Agent`**로 표기합니다.
*   **표준 상태 용어**: 프로세스 상태 전이에 대해 아래 용어를 고수합니다.
    *   **기동 (Startup)**: 시작, 가동 대신 사용
    *   **종료 (Shutdown)**: 정지, 중지, 중단 대신 사용
    *   **점검 (Check)**: 상태 확인, 검증 대신 사용
    *   **전환 (Switchover)**: 재해복구 Failover 대신 사용
*   **제목 및 항목 표기**: 국문 제목 옆에 영문을 괄호로 병기하는 방식(예: `기동 (Startup)`)은 지양하고 **국문 단독 표기**를 원칙으로 합니다.
*   **워크플로우 계층**: `시스템 워크플로우(서버 단위)` → `그룹 워크플로우(업무 단위)` → `시나리오 워크플로우(전체/전사 단위)` 단계를 유지합니다.
*   **시나리오 범주**: 시나리오 워크플로우는 **운영 자동화(IPL, Batch)**와 **재해복구 자동화(DR)**를 모두 포함합니다.

### **5. Agent 실행 환경 (Java Requirements)**
*   **JRE 포함**: Linux 및 Windows용 설치 패키지에는 Java 8(JRE)이 기본 포함됩니다.
*   **AIX 환경 주의**: AIX는 JRE가 포함되지 않으므로 설치 전 **Java 버전 확인 및 `JAVA_HOME` 설정**이 필수입니다.
*   **Java 버전 범위**: Java 8~11 사이를 공식 지원합니다.
*   **레거시 대응**: Java 1.8 미만 환경은 **Agent 1.2.10** 버전을 설치하며, 이 경우 실시간 로그 출력 기능이 제한됨을 명시합니다.

### **6. 컴포넌트 설계 표준 (3-Step Logic)**
모든 컴포넌트 템플릿은 **사전 검증(Pre-check) → 작업 실행(Execute) → 사후 확인(Post-check)**의 3단계 로직을 준수하여 작성합니다.

### **7. 다이어그램 색상 표준 (Mermaid Color Palette)**
다이어그램의 가독성과 일관성을 위해 아래의 Material Design 파스텔 컬러 코드를 사용합니다.

| 계층/상태 | 배경색 (Fill) | 테두리/텍스트 (Stroke) | 용도 |
|:---:|:---:|:---:|:---|
| **시스템 (System)** | `#e1f5fe` | `#01579b` | 단위 서버 작업, 하위 단계 |
| **그룹 (Group)** | `#fff9c4` | `#fbc02d` | 업무 단위, 중간 단계, 재해 판단 |
| **시나리오 (Scenario)** | `#ffebee` | `#c62828` | 전사 단위, 최상위 단계, 재해 발생 |
| **성공/완료 (Success)** | `#e8f5e9` | `#2e7d32` | 작업 완료, 정상 상태 |

*   **강조**: 우리 솔루션(MDRM)이나 최종 완료 노드는 `stroke-width:4px` 및 `font-weight:bold`를 추가하여 강조합니다.
*   **노드 모양**: 
    *   기본 프로세스: `[ ]`
    *   시작/종료 노드: `([ ])`
    *   판단/분기 노드: `{ }`
    *   **링크형 컴포넌트(하위 WF 호출)**: `[[ ]]` (이중 테두리)

### **8. 터미널 및 빌드 규칙 (Mandatory)**
*   **명령 실행 금지**: AI는 어떠한 터미널 명령도 직접 실행하지 않습니다. (예: `mkdocs build`, `mkdocs serve`, `run_doc` 등)
*   **사용자 주도**: 빌드, 서버 실행, Git 명령 등 모든 터미널 작업은 **사용자가 직접 수행**하는 것을 원칙으로 합니다. AI가 이를 임의로 시도하는 것은 가이드라인 위반입니다.
*   **수동 배포 원칙**: GitHub Push 및 배포 명령 또한 사용자의 명시적 요청과 실행을 통해서만 이루어집니다.

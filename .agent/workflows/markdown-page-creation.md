MDRM 교육 매뉴얼의 신규 페이지를 제작하거나 기존 페이지를 수정할 때 준수해야 하는 표준 스타일 가이드입니다. 이 워크플로우는 모든 페이지의 디자인 일관성과 프리미엄 품질을 유지하기 위해 고안되었습니다.

### **1. 기본 페이지 구조**

모든 마크다운(.md) 파일은 다음 순서를 따릅니다.

1.  **메인 제목**: `# 🚀 페이지 제목` (적절한 이모지 사용)
2.  **페이지 요약 박스**: 
    ```markdown
    !!! info "요약/목표"
        이 페이지에서 다루는 핵심 내용과 학습 목표를 친절한 어투(~합니다)로 작성합니다.
    ```
3.  **본문 섹션**: `## **1. 섹션명**` 형태로 번호와 굵은 글씨를 조합합니다.
4.  **다음 단계 카드**: 페이지 최하단에 배치합니다.
    ```markdown
    <div class="next-step-card-container" markdown>
    <a href="../대상_파일_경로/" class="next-step-card">
        <span class="next-content">
            <span class="next-step-label">Next Step</span>
            <span class="next-step-title">다음 페이지 제목</span>
        </span>
        <span class="next-step-icon">→</span>
    </a>
    </div>
    ```

### **2. 스타일 및 어투 원칙**

*   **톤앤매너**: 반드시 **경어체(~합니다, ~입니다)**를 사용합니다. 엔지니어에게 설명하는 친절하고 전문적인 어투를 유지합니다.
*   **시각 효과**: 
    *   중요 규칙: `!!! warning` 또는 `!!! danger` 사용
    *   팁/참고: `!!! tip` 사용
    *   흐름도: `mermaid` 다이어그램 적극 활용 (`flowchart LR` 등)
*   **표(Table)**: 테이블 헤더는 굵게 처리하며, 정렬(Align)을 통해 가독성을 확보합니다.
*   **이미지**: `../assets/images/` 경로를 사용하며, `{ width="100%" }` 등 속성을 부여합니다.

### **3. 파일 생성 및 반영 절차**

1.  **경로 선정**: `docs/day1/`, `docs/day2/`, `docs/day3/`, `docs/troubleshooting/` 중 성격에 맞는 폴더 내에 생성합니다.
2.  **파일명**: `MDRM_주제명.md` 또는 영문 파일명을 사용하며, 가급적 기존 파일들과 명명 규칙을 맞춥니다.
3.  **내비게이션 등록**: `mkdocs.yml` 파일의 `nav` 섹션에 적절한 들여쓰기로 새 페이지를 등록합니다.
4.  **검증**: `mkdocs build`를 수행하여 링크 오류나 문법 오류가 없는지 확인합니다.

### **4. 트러블슈팅 페이지 특화**

*   트러블슈팅 페이지는 `!!! danger` 박스로 시작하며, **현상 -> 원인 -> 해결 방법**의 논리적 구조를 유지합니다.
*   Next Step 카드 대신 `Troubleshooting 01`, `Troubleshooting End` 등의 라벨을 사용합니다.
